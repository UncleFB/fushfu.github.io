Get Started

通过 rustup 安装

Rustup 是 Rust 的安装以及版本管理工具

Linux 或 Mac 一键安装 curl --proto '=https' --tlsv1.2 https://sh.rustup.rs -sSf | sh

查看版本 rustc --version

更新 rustup update

卸载 rustup self uninstall

Cargo

cargo 是 rust 的构建工具和包管理器

通过 cargo --version 判断是否安装成功

通过 cargo new hello_cargo 即可新建 rust 项目

Cargo.toml 文件是 rust 的项目的配置文件，src 目录放置主要的源码

执行 cargo build 编译输出二进制可执行文件，目录为 target/debug

或者通过 cargo run 执行编译以及运行

cargo check 可以检查代码是否存在编译问题，但不输出可执行文件，因此效率更高，执行更快

如果想生成release版本，则执行 cargo build --release 此命令会在编译时进行优化，从而使代码运行的更快，可执行文件会输出到 target/release 目录下（不加 –release 参数时不会进行编译优化，输出目录为 target/debug）

fn main() {
    let mut count = 0;
    'counting_up: loop {
        println!("count = {count}");
        let mut remaining = 10;
        loop {
            println!("remaining = {remaining}");
            if remaining == 9 {
                //break 这一层 loop
                break;
            }
            if count == 2 {
                //break counting_up 标签的 loop
                break 'counting_up;
            }
            remaining -= 1;
        }
        count += 1;
    }
    println!("End count = {count}");
}

所有权

rust 的所有权是与其他编程语言最不同的地方，它保证了内存安全，也是 rust 无需垃圾收集器的关键

所有权的规则：

  • rust 中的每个值都有所有者
  • 同时只能有一个所有者
  • 当所有者超出作用域，值会被丢弃

如果是基本数据类型，也就是分配在栈上的变量。如下代码是没问题的，相当于拷贝了一个5到y上

1
2
3
4

    let x = 5;
    let y = x;

    println!("x = {}, y = {}", x, y);

但如果是引用类型，则会编译报错，下面代码中，相当于把字符串的所有权转移给了s2，那么s1就已经失效了

1
2
3
4

    let s1 = String::from("hello");
    let s2 = s1;

    println!("{}, world!", s1);

fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' {
            return &s[0..i];
        }
    }

    &s[..]
}

struct User {
    active: bool,
    username: String,
    email: String,
    sign_in_count: u64,
}
//不可变
fn main() {
    let user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };
}
//可变
fn main() {
    let mut user1 = User {
        active: true,
        username: String::from("someusername123"),
        email: String::from("someone@example.com"),
        sign_in_count: 1,
    };

    user1.email = String::from("anotheremail@example.com");
}
//active和sign_in_count为默认属性
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username: username,
        email: email,
        sign_in_count: 1,
    }
}
//参数及属性同名简写
fn build_user(email: String, username: String) -> User {
    User {
        active: true,
        username,
        email,
        sign_in_count: 1,
    }
}
//除email字段以外，其他属性使用user1的属性
fn main() {
    // --snip--

    let user2 = User {
        email: String::from("another@example.com"),
        ..user1
